# SWOT Analysis – Agile Methodologies, exemplu Scrum

## Context

Analiza se referă la utilizarea metodologiilor Agile, în special Scrum, pentru dezvoltarea aplicației ClinicQueue. Proiectul poate fi dezvoltat incremental, pornind de la funcționalități de bază și adăugând treptat îmbunătățiri.

## SWOT

| Strengths / Puncte tari | Weaknesses / Puncte slabe |
|---|---|
| Adaptabilitate ridicată la schimbări de cerințe. | Necesită implicare constantă din partea clientului sau Product Ownerului. |
| Livrare incrementală: aplicația poate avea rapid un MVP funcțional. | Estimările de timp și cost pot fi mai dificile la început. |
| Feedback frecvent de la utilizatori, ceea ce reduce riscul de produs nepotrivit. | Dacă backlog-ul nu este bine administrat, pot apărea priorități instabile. |
| Testarea poate fi integrată în fiecare sprint. | Echipa trebuie să aibă disciplină și comunicare bună. |
| Scope-ul poate fi ajustat în funcție de valoarea pentru business. | Documentația poate fi insuficientă dacă echipa se concentrează doar pe implementare. |

| Opportunities / Oportunități | Threats / Amenințări |
|---|---|
| Se pot adăuga rapid funcționalități noi, precum notificări, rapoarte sau autentificare. | Schimbările frecvente cerute de client pot produce scope creep. |
| Produsul poate fi testat devreme de utilizatori reali. | Lipsa disponibilității clientului poate încetini luarea deciziilor. |
| Sprinturile permit îmbunătățire continuă prin retrospective. | Dacă echipa nu respectă practicile Scrum, procesul poate deveni haotic. |
| Prioritizarea backlog-ului ajută echipa să livreze mai întâi funcționalitățile importante. | Dependințele externe, cum ar fi platforma CI sau infrastructura de deployment, pot genera întârzieri. |

## Concluzie

Agile/Scrum este potrivit pentru ClinicQueue deoarece aplicația poate evolua treptat. Mai întâi se poate implementa gestionarea programărilor, apoi căutarea, anularea, reprogramarea, notificările și rapoartele. Principalul risc este lipsa unui backlog clar și implicarea insuficientă a clientului.
