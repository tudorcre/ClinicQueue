# Project Timeline – ClinicQueue

| Etapă | Durată estimată | Activități | Livrabil |
|---|---:|---|---|
| 1. Inițiere | 1 zi | Alegerea temei, definirea scopului și a stakeholderilor. | Project charter |
| 2. Analiză | 1 zi | Identificarea cerințelor funcționale și non-funcționale. | Requirements document |
| 3. Planificare | 1 zi | Definirea backlog-ului, timeline-ului și riscurilor. | Timeline, backlog, risk log |
| 4. Setup tehnic | 1 zi | Creare repository, structură proiect, scripturi. | Repository inițial |
| 5. Implementare | 2 zile | Clasele Appointment, AppointmentService, interfață CLI. | Cod sursă funcțional |
| 6. Testare | 1 zi | Scriere și rulare teste unitare. | Test runner și rezultate teste |
| 7. CI/CD | 1 zi | Configurare GitHub Actions pentru build, test, analiză și deployment. | Workflow CI |
| 8. Finalizare | 1 zi | README, verificări finale, pregătire prezentare. | Proiect predabil |

## Milestones

| Milestone | Criteriu de finalizare |
|---|---|
| M1: Documentație inițială | Requirements, charter și risk log sunt completate. |
| M2: Funcționalitate de bază | Programările pot fi create, listate, căutate, anulate și reprogramate. |
| M3: Testare | Testele unitare trec local. |
| M4: CI | Pipeline-ul rulează automat la push. |
| M5: Deployment | Artefactul `.jar` este copiat în folderul `deployment/`. |
