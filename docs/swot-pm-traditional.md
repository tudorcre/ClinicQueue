# SWOT Analysis – Project Management tradițional

## Context

Analiza se referă la managementul tradițional de proiect, de tip Waterfall, aplicat unui proiect software precum ClinicQueue, o aplicație pentru gestionarea programărilor la un cabinet medical.

## SWOT

| Strengths / Puncte tari | Weaknesses / Puncte slabe |
|---|---|
| Structură clară a proiectului, cu etape definite: analiză, proiectare, implementare, testare și livrare. | Flexibilitate redusă când apar schimbări de cerințe în timpul proiectului. |
| Documentație detaliată încă de la început, utilă pentru cerințe, planificare și responsabilități. | Feedback-ul clientului apare de obicei târziu, după implementarea unei părți mari din produs. |
| Control bun al termenelor și bugetului dacă cerințele sunt stabile. | Dacă cerințele inițiale sunt incomplete, erorile sunt descoperite târziu și costă mai mult. |
| Potrivit pentru proiecte în care clientul știe exact ce dorește. | Testarea este adesea concentrată spre final, ceea ce crește riscul de defecte majore. |
| Scope-ul este clar definit la început și poate fi urmărit mai ușor. | Managementul requirement-urilor poate deveni rigid și birocratic. |

| Opportunities / Oportunități | Threats / Amenințări |
|---|---|
| Poate fi folosit în proiecte cu contracte fixe și livrabile clar definite. | Modificările legislative, tehnologice sau organizaționale pot face cerințele inițiale depășite. |
| Este ușor de înțeles de către managementul non-tehnic. | Clientul poate realiza târziu că produsul nu răspunde complet nevoilor reale. |
| Permite estimări inițiale clare pentru timp, cost și resurse. | Concurența sau schimbările pieței pot cere lansări mai rapide decât permite modelul tradițional. |
| Poate fi integrat cu proceduri organizaționale formale. | Întârzierile dintr-o etapă pot bloca toate etapele următoare. |

## Concluzie

Project Managementul tradițional este potrivit când cerințele sunt stabile, domeniul este bine înțeles și clientul preferă planificare predictivă. Pentru proiectul ClinicQueue, abordarea tradițională ar fi utilă dacă funcționalitățile sunt stabilite complet de la început. Principalul dezavantaj este adaptabilitatea redusă în cazul schimbării cerințelor cabinetului.
